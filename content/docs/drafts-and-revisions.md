# Drafts and revisions

Writing is messy. Kumooo keeps the mess off the live site until you say go.

## Drafts

New posts and pages start as drafts unless you publish immediately.

- Drafts are editable in the dashboard.
- Drafts are not on the public site.
- Preview links still work when you need a signed look.

## Publish

Hit **Publish**. The renderer serves the new HTML and bumps the edge cache version so readers are not stuck on yesterday's page.

## Revisions

Every save keeps history. If you overwrite a good draft with a bad one, open revisions and roll back.

You do not need a plugin for this. It is part of the content API.

## Scheduled publish

If you set a schedule, the post stays off the public list until that time. Check the dashboard status if something did not appear when you expected.

## Tips

- Use drafts for WIP tutorials.
- Publish pages (About, Docs index) when the nav should include them.
- After a big content fix, hard-refresh once. Cache invalidation is automatic; browsers are not always polite.

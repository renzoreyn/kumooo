# Notes on default themes

April 3

Every platform picks a default look. Most pick "safe corporate blog" and call it neutral.

We alias `default` to haru on purpose. Existing sites shouldn't break when we add seasons. New sites shouldn't inherit a name that means nothing.

Defaults matter because most people never change them. If the default is harsh, the web gets harsher. If it's calm, more calm sites exist. That's not philosophy. It's product math.

Haru uses a paper background, a restrained accent, and a nav that doesn't compete with your headline. Dark mode isn't an afterthought. I toggled it on a train last week and the contrast still held.

When you migrate an old site, you don't want surprise layout shifts. Alias mapping buys us time to rename themes in the dashboard without breaking bookmarks. Your readers see the same URLs. You see four seasons in the picker.

Try writing one real post before you customize CSS. If the theme fights your sentences, switch seasons. If it disappears, you picked right. Defaults should feel invisible on day one and stay invisible on day three hundred.

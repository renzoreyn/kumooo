# Small observation: lists beat dashboards

April 18

I deleted a Notion board yesterday. Replaced it with a markdown file and a dated list on my site.

Not because Notion is bad. Because I was maintaining a dashboard for thoughts that only needed timestamps.

Kumooo's home view is a list on purpose. Newest first. Title, date, excerpt if you want one. No widgets. No "engagement" column. Readers aren't metrics here.

When I demo haru, I show four short posts and one about page. That's enough to see rhythm: how titles wrap, how dates align, how a long paragraph breathes.

Lists also age well. A dashboard from 2019 looks dated fast. A chronological index from 2019 looks like history. That's nicer for personal sites and for small teams who actually want to find old decisions.

If your content is mostly updates, lean into the list. Write like you're leaving breadcrumbs for future you. Future you will thank you when search works and the archive page isn't empty. Add tags if you need grouping. Skip the rest.

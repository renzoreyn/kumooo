# Release notes belong on the open web

Slack threads die. Notion pages go stale. Email archives are where facts go to hide.

Put release notes on your site. Public. Linkable. Cached at the edge.

Natsu makes short posts look intentional. Version number in the title. Three bullets in the body. Ship.

I do this for Kumooo itself. When something changes in the renderer, I want a URL I can paste in a GitHub issue. Not a screenshot of a dashboard.

You don't need a changelog product for a solo project. You need markdown, a theme that respects lists, and the habit of publishing when you merge.

Tag releases if you want filters later. Keep titles boring and searchable. `2026-01-09 cache bump` beats `Big performance win!!!` every time.

Start small. One note about what changed today. Repeat tomorrow. Your future self greps the archive faster than Slack search anyway.

Pin the latest release note in nav if you ship weekly. Remove the pin when it stales. Nav clutter is worse than no pin. Natsu cards make stale pins obvious, which is another reason to keep posts short and dated.

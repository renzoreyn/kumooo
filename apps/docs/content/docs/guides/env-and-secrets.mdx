---
title: Env and secrets
description: What goes in .env and what stays in Cloudflare.
---

Local Next.js apps read env from `.env.local`. Cloudflare Workers read secrets from Wrangler / the dashboard.

## Local

Create `.env.local` (never commit it):

```bash
# example
NEXT_PUBLIC_SITE_URL=http://localhost:3000
```

`NEXT_PUBLIC_*` is visible in the browser. Everything else stays server-only.

## Cloudflare

For OpenNext Workers:

```bash
pnpm exec wrangler secret put MY_SECRET
```

Or set secrets in the Cloudflare dashboard under your Worker.

## Rules of thumb

1. Do not put API keys in client components.
2. Do not commit `.env*` files with real values.
3. Prefer Wrangler secrets for production.

Hosting on kumooo (`kumooo.site`) will manage platform secrets for you; your app secrets still belong to your project.
